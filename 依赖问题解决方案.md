# 🚨 依赖问题快速解决方案

## 问题描述
当前遇到 `Module not found: Can't resolve 'next-auth/react'` 等依赖包找不到的错误。

## 🔧 立即解决步骤

### 步骤1：停止开发服务器
在终端中按 `Ctrl + C` 停止当前运行的 `npm run dev`

### 步骤2：强制清理缓存
```bash
# 删除 node_modules 和 package-lock.json
rmdir /s node_modules
del package-lock.json

# 或者在文件管理器中手动删除这两个文件/文件夹
```

### 步骤3：重新安装依赖
```bash
npm install
```

### 步骤4：安装缺失的依赖
```bash
npm install next-auth @next-auth/prisma-adapter prisma @prisma/client bcryptjs jsonwebtoken stripe @types/bcryptjs @types/jsonwebtoken
```

### 步骤5：重启开发服务器
```bash
npm run dev
```

## 🎯 临时解决方案（如果上述步骤失败）

如果依赖安装仍有问题，我们可以先恢复到基础版本，然后逐步添加用户系统：

### 选项A：回退到基础版本
暂时移除用户认证相关的代码，专注于核心ASMR视频生成功能。

### 选项B：使用简化的用户系统
使用本地存储 + 简单状态管理，不依赖数据库，先实现基本的用户体验。

1. **检查Node.js版本**：确保使用Node.js 18+
2. **网络问题**：尝试使用npm镜像：`npm config set registry https://registry.npmmirror.com`
3. **权限问题**：以管理员身份运行命令行工具
4. **文件锁定**：确保没有其他程序占用项目文件
